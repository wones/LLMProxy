# LLM Proxy Service - 零依赖使用说明

## 项目简介

**LLM Proxy Service** 是一个轻量级的 LLM API 代理服务，支持 OpenAI 和 Anthropic 格式的 API，可以用于 Claude Code、OpenAI Codex、opencode 等工具的对接。

### 主要特性

- 🎯 **零依赖** - 打包后可直接运行，无需安装任何依赖
- 🔄 **格式兼容** - 同时支持 OpenAI 和 Anthropic 格式
- 📊 **日志记录** - 完整的请求和响应日志
- 📈 **统计分析** - Token 使用统计
- 🛠️ **工具调用** - 完整支持工具调用功能
- 🤖 **Claude Code 支持** - 可以对接 Claude Code
- 💻 **Codex 支持** - 可以对接 OpenAI Codex
- 🎨 **像素风格 UI** - 简洁美观的管理界面

## 配置管理

### 添加配置

1. 打开配置管理页面：http://localhost:8000/config
2. 点击 "添加配置" 按钮
3. 填写配置信息：
   - **配置名称**：如 `my-openai`
   - **目标 URL**：如 `https://api.openai.com/v1/chat/completions`
   - **API 格式**：选择 `openai` 或 `anthropic`
   - **App Key**：API Key（可选）
   - **App Secret**：API Secret（可选）
   - **是否流式响应**：开启或关闭

### CURL 导入

1. 复制现有的 cURL 命令
2. 点击 "从 CURL 导入" 按钮
3. 粘贴 cURL 命令
4. 系统自动解析并填充配置

### 激活配置

- 在配置列表中，点击配置行的 "启用" 按钮
- 激活后的配置会显示绿色高亮

## API 调用

### Claude Code 接入

```bash
# 设置环境变量
export ANTHROPIC_API_KEY=任意值
export ANTHROPIC_BASE_URL=http://localhost:8000/v1
```

### OpenAI Codex 接入

```bash
# 设置环境变量
export OPENAI_API_KEY=任意值
export OPENAI_BASE_URL=http://localhost:8000/v1
```

### cURL 示例

#### OpenAI 格式（非流式）

```bash
curl -X POST http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer dummy" \
  -d '{
    "model": "your-model-name",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

#### OpenAI 格式（流式）

```bash
curl -X POST http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer dummy" \
  -d '{
    "model": "your-model-name",
    "messages": [{"role": "user", "content": "你好"}],
    "stream": true
  }'
```

#### Anthropic 格式（非流式）

```bash
curl -X POST http://localhost:8000/v1/messages \
  -H "Content-Type: application/json" \
  -H "x-api-key: dummy" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "your-model-name",
    "messages": [{"role": "user", "content": "你好"}],
    "max_tokens": 1024
  }'
```

#### Anthropic 格式（流式）

```bash
curl -X POST http://localhost:8000/v1/messages \
  -H "Content-Type: application/json" \
  -H "x-api-key: dummy" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "your-model-name",
    "messages": [{"role": "user", "content": "你好"}],
    "max_tokens": 1024,
    "stream": true
  }'
```

## 流式响应说明

### 工作原理

流式响应由配置中的 "是否流式响应" 字段决定：

1. **配置优先**：首先使用配置中的 stream 值
2. **请求覆盖**：请求中的 stream 字段可以覆盖配置
3. **完整记录**：流式和非流式响应都会完整记录在日志中

### 配置说明

| 配置值 | 请求值 | 最终结果 |
|--------|--------|----------|
| true | 无 | 流式响应 |
| false | 无 | 非流式响应 |
| true | false | 非流式响应（覆盖） |
| false | true | 流式响应（覆盖） |

## 认证方式

### 支持的认证方式

| 方式 | 说明 |
|------|------|
| App-Key | 自定义 App Key，放在请求头中 |
| App-Secret | 自定义 App Secret，放在请求头中 |
| x-api-key | Anthropic 标准认证头 |
| Authorization | Bearer Token 认证 |

### 配置认证

在配置管理中填写 App Key 和 App Secret，系统会自动添加到请求头中。

## 常见问题

### Q: 如何添加新的 LLM 配置？

A: 在配置管理页面点击 "添加配置"，填写目标 URL、API 格式和认证信息，然后激活配置。

### Q: Claude Code 无法连接？

A: 检查配置是否激活，确保 ANTHROPIC_BASE_URL 设置为 http://localhost:8000/v1。

### Q: 工具调用不工作？

A: 确保配置中的 API 后端支持工具调用，并且工具定义正确。

### Q: 日志时间显示不正确？

A: 日志时间已修复为显示本地时区时间。

### Q: 如何查看完整的请求信息？

A: 在日志列表中点击时间，可以查看完整的请求头、请求内容和响应内容。
